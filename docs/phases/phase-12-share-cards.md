# Phase 12 — Share Cards

**Status:** Not started
**Depends on:** 08
**Scope:** S
**Recommended model:** `claude-sonnet-4-6` — OG image generation with Satori, token minting with RLS, and public share page. The Satori/JSX-to-SVG pattern has edge cases that benefit from Sonnet over Haiku.

## Goal

Turn an Ask answer into a shareable image — the viral hook. Brand-consistent, copy-heavy, designed for screenshots and Twitter/IG share.

## Deliverables

- `web/app/api/share-card/route.ts` — `POST /api/share-card` — generates PNG from an `ask_message_id`, stores to Supabase Storage, returns public URL.
- `web/app/share/[id]/page.tsx` — public HTML page rendering the shared answer with OG meta tags.
- `web/app/api/og/ask/[id]/route.ts` — Next.js Open Graph image route using `@vercel/og` / `next/og` ImageResponse.
- `web/components/ask/ShareButton.tsx` — copy link + download image buttons.
- `web/lib/sharing/renderCard.ts` — TSX template that produces the card (used by both PNG export and OG image).
- `web/supabase/migrations/004_share.sql` — `share_tokens` table (anonymous-accessible shared answers with expiry).
- `web/lib/sharing/tokens.ts` — mint/validate share tokens.

## Specification

### Card design

Single PNG, 1080×1350 (IG portrait) and 1200×630 (OG), same template rendered at different sizes.

Content layout:

```
────────────────────────────────────────
   ASTRI                [brand mark]
────────────────────────────────────────

   VERDICT
   ───────────────────────────────────
   {AskAnswer.verdict}   ← large type


   WHY
   ───────────────────────────────────
   • {why[0]}
   • {why[1]}
   • {why[2]}


   TIMING
   ───────────────────────────────────
   {timing.summary}


   ── based on {charts_used.join(' · ')} ──
        astri.app/share/{token}
────────────────────────────────────────
```

Dark palette, one serif for the verdict, one sans for everything else. No background noise.

### Public share page

`/share/[id]` renders the answer in HTML (same data, better accessibility than an image). OG image points at `/api/og/ask/[id]`. Twitter card = `summary_large_image`. The public page:

- Does NOT show the birth details or the user's name.
- Shows topic + tone used (chips).
- Shows `verdict`, `why`, `timing`, `confidence` — that's it. No `advice` (felt-too-personal), no full technical basis (teases only `charts_used`).

### Share tokens

Anonymous visitors must not be able to enumerate messages. `share_tokens` table:

```sql
create table public.share_tokens (
  id uuid primary key default gen_random_uuid(),
  ask_message_id uuid not null references public.ask_messages(id) on delete cascade,
  token text not null unique,
  created_by uuid not null references public.user_profiles(id) on delete cascade,
  expires_at timestamptz,                  -- null = never
  revoked_at timestamptz,
  created_at timestamptz not null default now()
);
-- RLS: owner can read/write/revoke their own tokens.
-- Public SELECT via an RPC that filters by token + checks expiry/revoked.
```

`/share/[token]` looks up the token via a Postgres RPC with `SECURITY DEFINER` that returns only the sanitized public payload. No service role on the client.

### Image generation

Use `next/og` (Edge runtime). `renderCard.ts` returns a JSX component; the OG route wraps it in `new ImageResponse(...)`. PNG download uses the same component rendered through a Node-side renderer (Satori + resvg) and uploaded to Supabase Storage at `share-cards/{token}.png`.

### Share button UX

On an `AnswerCard`, "Share" opens a sheet with:

- **Copy link** — copies `https://astri.app/share/{token}`.
- **Download image** — hits `/api/share-card`, triggers download.
- **Revoke** — destroys the token (available if one exists).

Share button is hidden on error states and on the last-assistant-turn while streaming.

### Moderation safety

Because shared content is rendered publicly, run a lightweight length + profanity check on `verdict` and `why` before minting a token. Anything flagged blocks sharing with "this answer can't be shared — try rephrasing your question".

## Acceptance criteria

- [ ] Clicking Share mints a token and returns a `/share/{token}` link.
- [ ] That link renders the sanitized public page (no PII, correct OG tags).
- [ ] Pasting the link into Twitter, Slack, or LinkedIn shows a preview image generated by `/api/og/ask/[id]`.
- [ ] Download image produces a 1080×1350 PNG stored under `share-cards/`.
- [ ] Revoking a token makes `/share/{token}` 404.
- [ ] Moderation blocks attempts to share content with flagged patterns.
- [ ] Typecheck + lint + tests pass.

## Out of scope

- Multi-answer "digest" share (post-MVP).
- Custom user-uploaded backgrounds (post-MVP).
- Watermark removal for premium (post-MVP if ever).

## Verification

1. Ask a question. Share. Copy link. Open in a new incognito tab → sanitized page renders.
2. Paste link into any OG-aware surface — preview image visible.
3. Download image button produces a valid PNG. Open it — copy is readable, no layout cut-off.
4. Revoke the token — link 404s.
